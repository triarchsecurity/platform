Where to start (by role)

  1. New developer / junior on the team — start with the Git & GitHub primer. 22 scenes, ~15 min, gets them past commits and merge strategies.
  2. Tech lead / security-curious engineer — start with the GitHub security primer. Covers 2FA, branch protection, OIDC, and the 8 hardening settings — plus the plan-tier reality.
  3. Engineering manager scoping a CI/CD project — read the CI/CD package landing, then open the executive overview.
  4. Engineer ready to actually ship — open the walkthrough, answer the 10-question form (or run discovery), download deploy.md, hand it to Claude Code.
  5. Sales / sponsor showing a prospect — three minutes of the CI/CD movie tells the story.
  6. Auditor / consultant inheriting a half-built pipeline — gap-analysis.md + deploy.md together produce a defensible state report and remediation plan in one Claude Code session.
