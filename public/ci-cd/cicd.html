From GitHub + a React app

Built for the actual SMB starting state: just GitHub and source code. No cloud account, no IDP, no database, no observability assumed. The framework maps what exists, names what doesn't, and provisions the rest — hardened ruleset, three protected environments, OIDC (zero static keys), signed builds with SLSA provenance, and security scanning on every gate.

Step 1 — Claude Code prompt. Runs against the customer's GitHub org/repo and produces a Triarch-branded HTML report scoring every framework item as pass / partial / fail / unknown. Includes explicit no-credential-logging protocol.

Step 2 — Claude Code prompt. Takes the gap-analysis findings and walks through implementing each gap. Uses the scaffold as templates. Never asks for or logs credentials — for every secret, prints the gh secret set command for the user to run themselves.

Where to start

  1. If a teammate is new to git itself — start them on the Git & GitHub primer. 22 scenes, ~5 minutes, gets them past commits and merge strategies.
  2. If your org is loose on GitHub security — show them the GitHub 101 + security movie. Covers 2FA, branch protection, push protection, OIDC, and the 8 hardening settings — plus the plan-tier reality.
  3. If you're showing a prospect or stakeholder the CI/CD framework — open the CI/CD movie. Five minutes, full picture.
  4. If you're scoping the work — read the overview and the framework markdown.
  5. If you're an engineer ready to assess — open the walkthrough. Either answer the 10-question form (fast, rough) or download discovery.sh + gap-analysis.md and hand them to Claude Code (slower, comprehensive). Either way produces findings.
  6. To act on findings — hand deploy.md to the same Claude Code session along with the findings. It walks through implementing each gap. Never asks for or logs your credentials.
