## Spacing Scale

Tailwind v4 utility classes map directly to the 8-point scale. Use these classes
exclusively — do not add ad-hoc pixel values.

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon-to-label gap, badge inner padding |
| sm | 8px | `p-2` / `gap-2` | Button inner padding (y-axis), row cell gap |
| md | 16px | `p-4` / `gap-4` | Card padding, form field spacing |
| lg | 24px | `p-6` / `gap-6` | Section vertical rhythm |
| xl | 32px | `p-8` / `gap-8` | Page outer padding (matches existing pages: `p-8`) |
| 2xl | 48px | `mb-12` | Major section breaks (not used in this phase) |

Exceptions:
- `px-3 py-2` (12px/8px) for secondary action buttons — mirrors existing
  `bg-teal-600` button in projects/page.tsx line 131.
- Table row minimum touch target: 44px height via `py-3` + `text-sm` line height.


### 3. Add Member Form Card

A `<div className="mb-6 p-4 rounded-lg bg-zinc-900 border border-zinc-800">`.
Single-row layout on desktop: email input (flex-1) + role select + Add button.
Stack vertically on narrow viewports (below 640px) via `flex flex-col sm:flex-row gap-2`.

**Email input:**
```tsx
<input
  type="email"
  value={email}
  onChange={...}
  placeholder="customer@example.com"
  className="flex-1 px-3 py-2 bg-zinc-900 border border-zinc-700 rounded-md
             text-sm text-zinc-200 focus:outline-none focus:border-teal-500
             disabled:opacity-50"
  disabled={adding}
/>
```
Pattern: `new/page.tsx` lines 188–190 (identical classes).

Client-side validation: `email.includes('@') && email.includes('.')` before
submit. On invalid format, disable the Add button and show helper text
`"Enter a valid email address"` as `text-[10px] text-red-400 mt-1` below the
input. Do not submit to the server with a malformed email.

**Role select:**
```tsx
<select
  value={role}
  onChange={...}
  className="px-3 py-2 bg-zinc-900 border border-zinc-700 rounded-md
             text-sm text-zinc-200 focus:outline-none focus:border-teal-500
             disabled:opacity-50"
  disabled={adding}
>
  <option value="admin">admin</option>
  <option value="viewer">viewer</option>
</select>
```
Default selection: `viewer`. Options lowercase, no capitalisation. Width:
`w-28` (shrink-0). No `staff` option — staff is set via SQL.

**Add button (primary CTA):**
```tsx
<button
  onClick={handleAdd}
  disabled={!isEmailValid || adding}
  className="flex items-center gap-1.5 px-3 py-2 text-sm bg-teal-600 text-white
             rounded-md hover:bg-teal-500 disabled:opacity-50 flex-shrink-0"
>
  {adding ? <Loader2 size={14} className="animate-spin" /> : <UserPlus size={14} />}
  Add Member
</button>
```
Icon: `UserPlus` (lucide). Loading: `Loader2 animate-spin` replaces icon during
the POST, matching the pattern in `new/page.tsx` line 225.

Shown after add or remove completes. Replaces any previous banner.
Auto-dismisses after 4 seconds OR via the X button.

**Success:**
```tsx
<div className="mb-4 p-4 rounded-md border text-sm

  <div className="flex items-center justify-between">
    <span>{message}</span>
    <button onClick={dismiss}><X size={14} /></button>

```

**Error:**
```tsx
<div className="mb-4 p-3 rounded-md bg-red-500/10 border border-red-500/20
                text-sm text-red-400 flex items-center gap-2">
  <AlertCircle size={14} /> {errorMessage}
</div>
```

Pattern: `projects/page.tsx` lines 137–148, `new/page.tsx` lines 175–178.

### 5. Members Table Card

A `<div className="rounded-lg border border-zinc-800 bg-zinc-900/50 overflow-hidden">`.

**Table layout:** `<table className="w-full text-sm">` with `<thead>` and `<tbody>`.
Use a table (not cards) — the data is columnar (email, role, joined date, action)
and scanning across rows is the primary interaction. Cards would waste vertical
space for what is a list-management surface.

**Column headers (`<th>`):**
```
text-[10px] text-zinc-500 uppercase tracking-wide text-left px-4 py-2 border-b border-zinc-800
```
Columns: Email | Role | Joined | (empty — actions column, right-aligned)

**Rows (`<tr>`):**
```
border-b border-zinc-800 last:border-0 hover:bg-zinc-800/30
```

**Cells (`<td>`):**
```
px-4 py-3 text-zinc-200
```

**Email cell:** `font-mono text-[10px]` — preserves display casing from DB.

**Role cell:** A small badge. Use existing badge pattern from `projects/page.tsx`
lines 177–179:
```tsx
<span className={`px-1.5 py-0.5 rounded text-[10px] border ${ROLE_COLORS[member.role]}`}>
  {member.role}
</span>
```
Role color map:
```ts
const ROLE_COLORS = {
  admin:  'bg-teal-500/20 text-teal-400 border-teal-500/30',
  viewer: 'bg-zinc-700 text-zinc-400 border-zinc-600',
  staff:  'bg-amber-500/20 text-amber-400 border-amber-500/30',
};
```
Staff rows are read-only (no remove button) — staff membership is managed via
SQL per the decisions in CONTEXT.md.

**Joined date cell:** `text-zinc-500 text-xs`. Format: `MMM D, YYYY`
(e.g. "May 3, 2026"). Use `new Date(member.createdAt).toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })`.

**Remove action cell:** Right-aligned. Render only for non-staff rows.
```tsx
<td className="px-4 py-3 text-right">
  <button
    onClick={() => handleRemove(member)}
    className="text-zinc-600 hover:text-red-400 transition-colors"
    aria-label={`Remove ${member.email}`}
  >
    <Trash2 size={14} />
  </button>
</td>
```
No text label — icon only. `aria-label` for accessibility. Pattern: `projects/page.tsx` line 289.

**Loading state (skeleton):** While the initial member list is fetching, render
3 skeleton rows:
```tsx
{[...Array(3)].map((_, i) => (
  <tr key={i} className="border-b border-zinc-800">
    <td className="px-4 py-3"><div className="h-4 w-48 bg-zinc-800/50 rounded animate-pulse" /></td>
    <td className="px-4 py-3"><div className="h-4 w-16 bg-zinc-800/50 rounded animate-pulse" /></td>
    <td className="px-4 py-3"><div className="h-4 w-24 bg-zinc-800/50 rounded animate-pulse" /></td>
    <td className="px-4 py-3" />
  </tr>
))}
```
Pattern: `projects/page.tsx` lines 153–155.

```tsx
<tr>
```

## Interaction Contracts

### Adding a member — happy path

1. Staff enters email + selects role + clicks "Add Member".
2. Button disables, icon swaps to `Loader2 animate-spin`.
3. `POST /api/admin/projects/{key}/members` with `{ email, role }`.
4. On 201: add row to local state optimistically (no full refetch), show success
   banner "member@example.com added as viewer.", reset form.
5. Banner auto-dismisses after 4 seconds.

### Adding a duplicate email (409 conflict)

Server returns 409. UI shows the error banner:
`"member@example.com is already a member of this project."`
Form email field remains populated so staff can correct if needed (e.g. was a
typo). Role field resets to `viewer`.

### Malformed email (client-side)

The Add button is disabled when `email` does not contain `@` and `.` after the
`@`. Below the email input, show `text-[10px] text-red-400 mt-1`:
`"Enter a valid email address"`
No server request is made.

### Removing a non-self member

1. Staff clicks the `Trash2` icon on any non-staff row.
2. `DELETE /api/admin/projects/{key}/members/{email}`.
3. On 200: remove row from local state, show success banner:
   `"member@example.com removed."`

No confirmation modal for standard remove. The operation is low-stakes for a
staff-only tool; the backfill and hard-delete semantics are already locked in
CONTEXT.md.

### Removing yourself (staff removing own staff row)

Staff rows have no Remove button — they are rendered read-only. The current user
cannot remove their own staff membership from this page. Staff membership is
managed via SQL only. This sidesteps the self-removal edge case entirely: the UI
simply does not surface the action.

### Server error (5xx or network failure)

Show error banner: `"Something went wrong. Please try again."`
Re-enable the Add button. Do not clear the form.

### Loading states during add/remove

- Add: button disabled + spinner icon. Table rows remain interactive.
- Remove: `Trash2` icon on that specific row replaced by `Loader2 animate-spin`
  (size 14). Other rows remain interactive.

## Component Decomposition

| Component | Type | File path | Notes |
|-----------|------|-----------|-------|
| `MembersPage` | Server component | `src/app/admin/platform/projects/[key]/members/page.tsx` | Staff guard server-side, fetches initial member list |
| `MembersClient` | Client component (`'use client'`) | `src/app/admin/platform/projects/[key]/members/MembersClient.tsx` | Add form + table + feedback banner state |

and passes `projectName`, `projectKey`, and initial `members[]` as props to
`MembersClient`. The client component owns all mutation state.

`projects/page.tsx` that links to the members page uses `px-2.5 py-1.5 text-xs`
to match the existing provisioning button pattern at lines 263–288. This is a
code-level implementation detail that mirrors existing buttons in that file — it
is not a declared spacing contract value. The Spacing Scale contract above
governs new spacing decisions on the members page itself.

- `Users`, `UserPlus`, `Trash2`, `Loader2`, `X`, `AlertCircle` from `lucide-react`
