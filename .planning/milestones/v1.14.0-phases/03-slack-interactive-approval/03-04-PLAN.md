<objective>
The signature-verified Slack callback that lets staff promote (or reject) a release from inside Slack.

Purpose: GATE-09 (HMAC verify + 5-min replay window), GATE-09a (resolve release + identity-mapped staff actor), and consumer of GATE-08 (verify embedded payload signature). This route is the ONLY new endpoint in Phase 3. It accepts Slack's URL-encoded `payload` form field, verifies the request came from Slack (signature header), verifies the button value wasn't tampered with (embedded signature), maps the Slack user to a staff email, and dispatches to the shared release-actions helpers from Plan 03-02.

Output: New `src/app/api/slack/interact/route.ts` (POST handler) plus a Vitest suite covering signature verification paths, payload verification paths, the identity dispatch table, and stale-message handling.

<!-- Slack interactive payload shape (from Slack docs + existing bug-action route) -->
Body is `application/x-www-form-urlencoded` with a single field `payload` whose value is a JSON string with shape:
```typescript
{
  type: 'block_actions',
  user: { id: string, name: string, username?: string },
  actions: Array<{ action_id: string, block_id: string, value: string, ... }>,
  response_url: string,
  message: { ts: string, ... },
  channel: { id: string, name?: string },
}
```

action_ids the handler MUST dispatch on (set by plan 03-03 in slack.ts):
- 'slack_promote' → approveRelease (sets status='approved'; Phase 4 will add prod-promotion afterwards, but for Phase 3 the button is the customer-side approval flow's mirror)
- 'slack_reject' → rejectRelease

action.value is the packed signed payload `${releaseId}.${nonce}.${sig}` from signPayload.

<!-- response_url usage -->
After verifying & dispatching, return JSON with `{ replace_original: true, text, blocks }` to update the original message in-channel.
For ephemeral errors (unmapped user, stale state), return `{ response_type: 'ephemeral', replace_original: false, text }`.

    Handler flow (in order):
    1. Read raw body via `req.text()` ONCE. Store in `rawBody`.
    2. Read `X-Slack-Request-Timestamp` and `X-Slack-Signature` headers.
    3. Call `verifySlackSignature({ rawBody, timestamp, signature })`. On `{ ok: false }`, return 401 with `{ error: <reason> }`. (no_secret in prod is a misconfig — return 401, not 500, to keep the surface flat.)
    4. Parse rawBody as URLSearchParams, extract `payload` field, JSON.parse it. On parse failure → 400 `{ error: 'malformed_payload' }`.
    5. Validate payload.type === 'block_actions' and payload.actions[0] exists. Else 400 `{ error: 'unsupported_payload' }`.
    6. Extract `actionId = action.action_id`, `packedValue = action.value`, `slackUserId = payload.user.id`.
    7. Dispatch table:
       - actionId === 'slack_promote' → expectedAction = 'promote'
       - actionId === 'slack_reject'  → expectedAction = 'reject'
       - else → 200 with ephemeral `{ response_type: 'ephemeral', replace_original: false, text: 'Unknown action' }`.
    8. `verifyPayload(packedValue, expectedAction)`. On `{ ok: false }`, return 401 `{ error: 'invalid_payload_signature' }`.
    9. Resolve staff email: `email = resolveSlackUserEmail(slackUserId)`. If null → 200 ephemeral `{ response_type: 'ephemeral', replace_original: false, text: 'Your Slack user is not mapped to a staff account. Contact admin.' }` and `console.warn('[slack-interact] unmapped user', { slackUserId })`. No DB write.
    10. Look up release by id (from verifyPayload result). If not found → 200 ephemeral `{ ..., text: 'Release not found' }`.
    11. Stale-message guard: if `release.status` is already `'approved'` and actionId === 'slack_promote' → return 200 with replace_original:true updating the message to "Already promoted" + ephemeral note "Already approved by {existing approver email} on {date}". Same for `'rejected'` + 'slack_reject'. (Use `releaseApprovals` to fetch the most recent decision row for the message.)

    13. Dispatch:
        - 'slack_promote' → `approveRelease({ release, approverEmail: email, ipAddress, userAgent })`
        - 'slack_reject'  → `rejectRelease({ release, approverEmail: email, reason: 'Rejected via Slack', ipAddress, userAgent })` (reason comes from a fixed string for v1.14 — CONTEXT.md Area 4 mentions reject "{reason excerpt 80 chars}" but the button click does not collect a fresh reason; using a constant is the v1.14 contract — improving this to a Slack modal is deferred.)
    14. On `{ ok: false }` from helpers (rare — would mean the stale guard missed something), return 200 ephemeral with the helper message.
    15. On `{ ok: true }`, return 200 with replace_original:true, text + blocks reflecting new state. Block 1 mrkdwn: `:white_check_mark: *Promoted to production by @{slackUsername}*` for promote, or `:x: *Rejected by @{slackUsername}*` for reject.    
    ```ts
    import { NextRequest, NextResponse } from 'next/server';
    import { db } from '@/lib/db';
    import { releaseLogs, releaseApprovals } from '@/db/schema';
    import { and, desc, eq } from 'drizzle-orm';
    import { verifySlackSignature, verifyPayload } from '@/lib/slack-crypto';
    import { resolveSlackUserEmail } from '@/lib/slack-identity';
    import { approveRelease, rejectRelease } from '@/lib/release-actions';

    type ActionId = 'slack_promote' | 'slack_reject';
    const ACTION_TO_EXPECTED: Record<ActionId, 'promote' | 'reject'> = {
      slack_promote: 'promote',
      slack_reject: 'reject',
    };

    export async function POST(req: NextRequest) {
      const rawBody = await req.text();
      const timestamp = req.headers.get('x-slack-request-timestamp');
      const signature = req.headers.get('x-slack-signature');

      const sig = verifySlackSignature({ rawBody, timestamp, signature });
      if (!sig.ok) {
        return NextResponse.json({ error: sig.reason }, { status: 401 });
      }

      const params = new URLSearchParams(rawBody);
      const payloadStr = params.get('payload');
      if (!payloadStr) return NextResponse.json({ error: 'no_payload' }, { status: 400 });

      let payload: any;
      try { payload = JSON.parse(payloadStr); }
      catch { return NextResponse.json({ error: 'malformed_payload' }, { status: 400 }); }

      if (payload?.type !== 'block_actions' || !Array.isArray(payload.actions) || !payload.actions[0]) {
        return NextResponse.json({ error: 'unsupported_payload' }, { status: 400 });
      }

      const action = payload.actions[0];
      const actionId = action.action_id as string;
      const packedValue = action.value as string;
      const slackUserId = payload.user?.id as string | undefined;
      const slackUserName = (payload.user?.username ?? payload.user?.name ?? 'someone') as string;

      if (actionId !== 'slack_promote' && actionId !== 'slack_reject') {
        return NextResponse.json({
          response_type: 'ephemeral',
          replace_original: false,
          text: 'Unknown action',
        });
      }

      const expected = ACTION_TO_EXPECTED[actionId as ActionId];
      const verifiedPayload = verifyPayload(packedValue, expected);
      if (!verifiedPayload.ok) {
        return NextResponse.json({ error: 'invalid_payload_signature' }, { status: 401 });
      }

      const email = resolveSlackUserEmail(slackUserId);
      if (!email) {
        console.warn('[slack-interact] unmapped user', { slackUserId });
        return NextResponse.json({
          response_type: 'ephemeral',
          replace_original: false,
          text: 'Your Slack user is not mapped to a staff account. Contact admin.',
        });
      }

      const [release] = await db
        .select()
        .from(releaseLogs)
        .where(eq(releaseLogs.id, verifiedPayload.releaseId));

      if (!release) {
        return NextResponse.json({
          response_type: 'ephemeral',
          replace_original: false,
          text: 'Release not found',
        });
      }

      // Stale-message guard
      const wantsApprove = actionId === 'slack_promote';
      const wantsReject = actionId === 'slack_reject';
      if ((wantsApprove && release.status === 'approved') || (wantsReject && release.status === 'rejected')) {
        const [existing] = await db
          .select()
          .from(releaseApprovals)
          .where(and(eq(releaseApprovals.releaseId, release.id), eq(releaseApprovals.decision, release.status!)))
          .orderBy(desc(releaseApprovals.approvedAt))
          .limit(1);

        const verb = wantsApprove ? 'promoted' : 'rejected';
        const date = existing?.approvedAt?.toISOString().slice(0, 10) ?? 'earlier';
        const actor = existing?.approverEmail ?? 'someone';
        return NextResponse.json({
          response_type: 'ephemeral',
          replace_original: true,
          text: `Already ${verb} by ${actor} on ${date}`,
          blocks: [
            {
              type: 'section',
              text: {
                type: 'mrkdwn',
                text: wantsApprove
                  ? `:white_check_mark: *Already promoted by ${actor} on ${date}*`
                  : `:x: *Already rejected by ${actor} on ${date}*`,
              },
            },
          ],
        });
      }

      const ipAddress = req.headers.get('x-forwarded-for')?.split(',')[0]?.trim() ?? null;
      const userAgent = req.headers.get('user-agent')?.slice(0, 512) ?? null;

      if (wantsApprove) {
        const result = await approveRelease({ release, approverEmail: email, ipAddress, userAgent });
        if (!result.ok) {
          return NextResponse.json({
            response_type: 'ephemeral',
            replace_original: false,
            text: result.message,
          });
        }
        return NextResponse.json({
          replace_original: true,
          text: `Promoted to production by @${slackUserName}`,
          blocks: [
            {
              type: 'section',
              text: { type: 'mrkdwn', text: `:white_check_mark: *Promoted to production by @${slackUserName}* (${email})` },
            },
          ],
        });
      }

      // wantsReject
      const result = await rejectRelease({
        release,
        approverEmail: email,
        reason: 'Rejected via Slack',
        ipAddress,
        userAgent,
      });
      if (!result.ok) {
        return NextResponse.json({
          response_type: 'ephemeral',
          replace_original: false,
          text: result.message,
        });
      }
      return NextResponse.json({
        replace_original: true,
        text: `Rejected by @${slackUserName}`,
        blocks: [
          {
            type: 'section',
            text: { type: 'mrkdwn', text: `:x: *Rejected by @${slackUserName}* (${email})` },
          },
        ],
      });
    }
    ```

    SECURITY CRITICAL POINTS (do NOT compromise on these — Plan checker will flag any deviation):
    - rawBody is read via `req.text()` ONCE before any parsing. Do not call `req.formData()` first — that consumes the stream and leaves nothing to verify.
    - rawBody is the EXACT byte string Slack signed; passing a re-stringified URLSearchParams is wrong.
    - Signature verify happens BEFORE any body interpretation; on failure, no DB lookups, no logs of payload contents.
    - Embedded payload verify happens BEFORE release lookup — a bad signature must not even hit the DB.
    - Unmapped Slack user happens BEFORE any DB-write helper call.
    - `approverEmail` is the resolved staff email from SLACK_USER_MAP — never the Slack username.
    - `expected` action string ('promote' / 'reject') MUST match the second arg passed to `signPayload` in plan 03-03's notifyReleaseApproved. Do not typo.

    ```ts
    function buildSignedRequest(rawBody: string, opts: { tsOffsetSec?: number; tamperSig?: boolean } = {}) {
      const ts = String(Math.floor(Date.now() / 1000) + (opts.tsOffsetSec ?? 0));
      const baseString = `v0:${ts}:${rawBody}`;
    ```
