## 2. Fix dev-portal CL-3 Violation (DB namespace) **[claude-do]**

Dev-portal currently has both `apphosting.yaml` and `apphosting.dev.yaml` pointing to the same `DATABASE_URL_PORTAL` secret. Dev writes leak to prod database.

What I'll do:
```bash
# 1. Read current portal secret
PORTAL_URL=$(firebase apphosting:secrets:access DATABASE_URL_PORTAL --project triarch-dev-website)

# 2. Confirm/create portal_dev database in same CRDB cluster
psql "$PORTAL_URL" -c "CREATE DATABASE IF NOT EXISTS portal_dev"

# 3. Clone schema (no data) from portal → portal_dev
pg_dump --schema-only "$PORTAL_URL" | psql "${PORTAL_URL%/portal*}/portal_dev"

# 4. Construct dev URL and mint new GCP secret
PORTAL_DEV_URL=$(echo $PORTAL_URL | sed 's|/portal\b|/portal_dev|')
echo "$PORTAL_DEV_URL" | firebase apphosting:secrets:set DATABASE_URL_PORTAL_DEV --project triarch-dev-website --data-file -

# 5. Grant FAH portal-dev backend access
firebase apphosting:secrets:grantaccess DATABASE_URL_PORTAL_DEV --backend portal-dev --project triarch-dev-website

# 6. Update dev-portal's apphosting.dev.yaml — change secret reference DATABASE_URL_PORTAL → DATABASE_URL_PORTAL_DEV (1-line diff, commit on Phase 29 branch)
```
