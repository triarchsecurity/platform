<objective>
Glue Plan 11-01's parser to the database. ```typescript
// Validates parsed refs against bug_reports / feature_requests and writes valid links.
// Returns the count of links created. Caller should not block on this — designed to be
// fire-and-forget from the ingest route, with try/catch around the call site.
//
// projectKey is needed to construct the external URL for bare `#N` refs from projects.github_repo.
export async function stampLinksFromCommit(input: {
  releaseId: string;        // uuid of the release_logs row just inserted
  commitMessage: string;    // raw commit message text
  projectKey: string;       // projects.key — used to look up github_repo for external URLs
}): Promise<{ stamped: number; dropped: number }>;
```

From src/db/schema.ts (releaseLogLinks columns and constraints):
- `release_id` UUID NOT NULL FK → release_logs.id
- `link_type` VARCHAR(16) NOT NULL — must be 'bug' | 'feature' | 'external' (CHECK constraint enforced by Phase 10 migration)
- `bug_id` UUID NULLABLE FK → bug_reports.id (CASCADE on delete)
- `feature_id` UUID NULLABLE FK → feature_requests.id
- `external_url` TEXT NULLABLE
- `source` VARCHAR(16) NOT NULL — 'commit' | 'manual' (validated app-side; Phase 11 always writes 'commit' from this stamper)
- Indexes: `release_id_idx`, `bug_id_idx`, `feature_id_idx`

From src/app/api/platform/ingest/release-logs/route.ts (existing ingest hook point):
- Body deserialized at lines 13–25 (commit message available via `entries` JSONB or — TBD at execution time — a top-level `commitMessage` / `summary` field)
- Insert at lines 47–62 returns `release` row with `release.id`
- Hook AFTER the INSERT, BEFORE the `NextResponse.json(release, { status: 201 })`
- The commit message source is the most uncertain field. Determine at execution time: check the body for `commitMessage`, else fall back to the `summary` field, else loop over `entries[]` and concatenate `description` + `commit_sha` lines.

From src/db/schema.ts projects table:
- `githubRepo: varchar('github_repo', { length: 256 })` — nullable; format expected `'owner/repo'` (e.g. `'MyAlterLego/triarchsecurity-admin'`)
- External URL construction: `https://github.com/${githubRepo}/issues/${N}` ONLY when `githubRepo` is non-null

    **Algorithm:**
    1. Call `parseCommitRefs(commitMessage)` — get `ParsedRef[]`.
    2. Early return `{stamped: 0, dropped: 0}` if list is empty.
    3. Bucket refs by type: `bugIds`, `featureIds`, `externalNumbers`.
    4. Lowercase + dedupe bugIds and featureIds.
    5. Batch validate bugIds: `db.select({id: bugReports.id}).from(bugReports).where(inArray(bugReports.id, bugIds))` → `validBugIds: Set<string>`.
    6. Batch validate featureIds: same pattern against `featureRequests`.
    7. If `externalNumbers.length > 0`, look up `projects.github_repo` for the projectKey; only emit external rows when github_repo is non-null.
    8. Build INSERT rows:
       - For each valid bug id: `{releaseId, linkType: 'bug', bugId: id, featureId: null, externalUrl: null, source: 'commit'}`
       - For each valid feature id: same shape with featureId
       - For each external N (only if github_repo present): `{releaseId, linkType: 'external', bugId: null, featureId: null, externalUrl: '${github_repo_base}/issues/${N}', source: 'commit'}` where github_repo_base = `https://github.com/${githubRepo}`
    9. If insertRows.length > 0: `db.insert(releaseLogLinks).values(insertRows)` (single batched INSERT — Drizzle supports array values).
    10. Return `{stamped: insertRows.length, dropped: parsedRefs.length - insertRows.length}`.

    **Error handling — must not throw:**
    Wrap the entire body in try/catch. On any caught error, log via `console.error('[link-stamper] failed', err)` and return `{stamped: 0, dropped: parsedRefs?.length ?? 0}`. Caller (ingest route) is also wrapping in try/catch as defense-in-depth, but the stamper itself MUST be forgiving — a release ingest is more important than a link.

    // LINK-02 / LINK-03: best-effort auto-stamp commit refs onto release_log_links.
    // Wrapped in try/catch so any stamper failure NEVER blocks a release ingest.
    // The stamper is itself forgiving (catches internal errors) but defense-in-depth here.
    try {
      // Build the message text the parser will scan. Prefer explicit commitMessage on the body,
      // fall back to summary, then synthesize from entries[] descriptions if neither present.
      const messageText: string = (() => {
        if (typeof body.commitMessage === 'string' && body.commitMessage.trim().length > 0) {
          return body.commitMessage;
        }
        if (typeof summary === 'string' && summary.trim().length > 0) {
          return summary;
        }
        if (Array.isArray(entries)) {
          return entries
            .map((e: { description?: string }) => (typeof e?.description === 'string' ? e.description : ''))
            .filter(Boolean)
            .join('\n');
        }
        return '';
      })();

      if (messageText.length > 0) {
        await stampLinksFromCommit({
          releaseId: release.id,
          commitMessage: messageText,
          projectKey: project!.key,
        });
      }
    } catch (err) {
      console.error('[ingest/release-logs] link stamping failed (non-blocking)', err);
    }
    ```
