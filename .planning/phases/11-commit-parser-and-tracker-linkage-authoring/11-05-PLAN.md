<objective>
Close the LinksClient mount-fetch gap so existing release_log_links rows (auto-stamped from commits in plan 11-03 and any prior manual adds) appear as chips when staff load /admin/modules/release-logs.

Purpose: Plan 11-04 shipped the LinksClient with optimistic add/remove and the GET/POST/DELETE API routes, but page.tsx passes initialLinks=[] and LinksClient never fetches on mount. As a result, existing links are invisible after every page reload — defeating half of LINK-04.

Output:
- LinksClient.tsx with a mount-keyed useEffect that calls GET /api/admin/release-logs/${releaseId}/links and hydrates state
- LinksClient.test.tsx with Vitest cases proving the fetch fires on mount, chips render from the response, and optimistic flow does not regress

Decision: Option A (client-side useEffect) chosen over Option B (augment platform GET). Rationale: A is one self-contained change in the component that already owns the chip state, reuses the existing /api/admin/release-logs/[id]/links endpoint that already augments titles, and avoids touching the membership-scoped /api/platform/release-logs route. B would force every release row to carry links[] in the list payload even when collapsed, and the platform route's membership-scoping would need extending to release_log_links.
</objective>

    2. Inside the `LinksClient` component, after the useState declarations, add:
       ```typescript
       useEffect(() => {
         // Skip mount fetch if parent already provided links — server-provided wins
         if (initialLinks.length > 0) return;
         let cancelled = false;
         (async () => {
           try {
             const res = await fetch(`/api/admin/release-logs/${releaseId}/links`);
             if (!res.ok) {
               console.error('[LinksClient] GET failed', res.status);
               return;
             }
             const data = (await res.json()) as { links: ReleaseLogLink[] };
             if (!cancelled && Array.isArray(data.links)) {
               setLinks(data.links);
             }
           } catch (err) {
             console.error('[LinksClient] GET error', err);
           }
         })();
         return () => { cancelled = true; };
       }, [releaseId, initialLinks.length]);
       ```
       Notes:
       - Dependency array uses `initialLinks.length` rather than `initialLinks` itself to avoid re-firing when parent re-renders the same array reference but with a new identity. The component is mounted per-release inside the expanded section, so this effect runs once per release expansion.
       - The `cancelled` flag prevents setState-after-unmount warnings if the user collapses the row mid-fetch.
       - DO NOT change the existing optimistic add/remove logic, props interface, or chip rendering.
