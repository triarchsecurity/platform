ParsedRef shape (this plan defines it):
```typescript
export type ParsedRef =
  | { type: 'bug'; id: string; source: 'commit' }       // id is full UUID
  | { type: 'feature'; id: string; source: 'commit' }   // id is full UUID
  | { type: 'external'; ref: string; source: 'commit' }; // ref is bare integer like '99' for `fixes #99`

export function parseCommitRefs(message: string): ParsedRef[];
```

UUID format (CockroachDB / Drizzle defaultRandom):
- 8-4-4-4-12 hex characters, lowercase or uppercase, separated by hyphens
- Regex fragment: `[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}` (case-insensitive `i` flag)

From src/db/schema.ts:
- `bug_reports.id` is `uuid('id').primaryKey().defaultRandom()` — full UUIDs
- `feature_requests.id` is `uuid('id').primaryKey().defaultRandom()` — full UUIDs
- These are NOT short integers; any short-integer match is therefore an external GitHub issue ref, not a Triarch bug/feature

    Create `src/lib/commit-parser.ts` implementing `parseCommitRefs(message: string): ParsedRef[]`. The regex MUST match exactly the 3 patterns defined in 11-CONTEXT decisions:

    **Pattern A — direct UUID refs (with optional `#` prefix):**
    ```typescript
    const DIRECT_BUG = /\b#?BUG-([0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12})\b/gi;
    const DIRECT_FEAT = /\b#?FEAT-([0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12})\b/gi;
    ```

    **Pattern B — verb-prefixed UUID refs:**
    ```typescript
    const VERB_BUG = /\b(?:closes?|fixes?|resolves?)\s+#?BUG-([0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12})\b/gi;
    const VERB_FEAT = /\b(?:closes?|fixes?|resolves?|implements?|adds?)\s+#?FEAT-([0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12})\b/gi;
    ```

    **Pattern C — verb-prefixed bare integer (external GitHub issue):**
    ```typescript
    const EXTERNAL_ISSUE = /\b(?:closes?|fixes?|resolves?)\s+#(\d+)\b/gi;
    ```

    **Implementation details:**
    - Pattern B fires BEFORE Pattern A so that `closes #BUG-{uuid}` is captured as a single ref (Pattern B), not double-counted as Pattern A.
    - Use a `Set<string>` (key: `${type}:${id|ref}`) for dedup.
    - Lowercase all UUIDs in output (canonical form for DB lookup).
    - `ParsedRef.source` is always `'commit'` for this parser. Manual links get `source='manual'` from a different code path (Plan 11-03 link mutation route).
    - Export both `parseCommitRefs` and the `ParsedRef` type.
    - File MUST NOT import anything from `@/lib/db`, `@/db/schema`, `next/server`, or any I/O module — pure function only. Verify via `grep -E "(from '@/lib/db'|from '@/db|from 'next)" src/lib/commit-parser.ts` returning empty.
