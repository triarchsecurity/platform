```typescript
// Strips Slack control characters that would trigger mentions or break message rendering.
// Idempotent: sanitizeForSlack(sanitizeForSlack(x)) === sanitizeForSlack(x).
export function sanitizeForSlack(text: string): string;

// Strips Unicode trickery (RTL override, zero-width chars) for safe display in admin UI.
// React already auto-escapes HTML; this layer is defense-in-depth against Unicode-based
// visual deception attacks (e.g., RTL override flipping URL display).
export function sanitizeForRender(text: string): string;
```

Slack control sequences to handle (from Slack mrkdwn docs + Pitfall 5):
- `<!channel>` `<!here>` `<!everyone>` — broadcast mentions
- `<@U[A-Z0-9]+>` — user mentions (e.g. `<@U123ABC>`)
- `<#C[A-Z0-9]+>` — channel mentions
- `<!subteam^S[A-Z0-9]+>` — usergroup mentions
- `<URL|text>` — Slack link syntax (replace with just `URL` or escape `<` to `‹`)

Unicode dangers (from research/PITFALLS.md Pitfall 5):
- `U+202E` RIGHT-TO-LEFT OVERRIDE — visually reverses subsequent text
- `U+202D` LEFT-TO-RIGHT OVERRIDE
- `U+200B` ZERO WIDTH SPACE
- `U+200C` ZERO WIDTH NON-JOINER
- `U+200D` ZERO WIDTH JOINER
- `U+FEFF` ZERO WIDTH NO-BREAK SPACE (BOM)

    ```typescript
    const SLACK_BROADCAST = /<!(channel|here|everyone)>/gi;
    const SLACK_USER_MENTION = /<@U[A-Z0-9]+>/gi;
    const SLACK_CHANNEL_MENTION = /<#C[A-Z0-9]+(\|[^>]*)?>/gi;
    const SLACK_SUBTEAM = /<!subteam\^S[A-Z0-9]+(\|[^>]*)?>/gi;
    const SLACK_LINK = /<(https?:\/\/[^|>\s]+)\|[^>]*>/gi;

    export function sanitizeForSlack(text: string): string {
      return text
        .replace(SLACK_LINK, '$1')
        .replace(SLACK_BROADCAST, '‹!$1›')
        .replace(SLACK_USER_MENTION, '‹@user›')
        .replace(SLACK_CHANNEL_MENTION, '‹#channel›')
        .replace(SLACK_SUBTEAM, '‹@group›');
    }
    ```

      return text.replace(ZERO_WIDTH_AND_OVERRIDE, '');
    }
