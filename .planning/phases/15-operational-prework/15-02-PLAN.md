<objective>
Create the GoDaddy DNS record for `portal.triarch.dev` pointing at the Firebase App Hosting target. This must land BEFORE the FAH `portal-prod` backend's custom domain wiring (Plan 15-04) so FAH's domain-validation step sees the correct DNS during its check.

Purpose: Without DNS, the FAH custom domain attachment fails with a validation error and `https://portal.triarch.dev` returns SERVFAIL/NXDOMAIN. Customers cannot reach the portal. Phase 25 cutover (admin → portal 301) cannot ship.
Output: GoDaddy A or CNAME record on `portal.triarch.dev` resolving to FAH target hostname. (Specific record type — A vs CNAME — is decided in Task 2 based on what FAH's custom-domain wizard publishes.)
</objective>

<reference>
Existing DNS records on triarch.dev (verifiable):
- `admin.triarch.dev` — already pointing at admin's FAH backend (precedent)
- Apex domain managed via GoDaddy — Mike has account access; `mcp__godaddy__*` MCP tools available

FAH custom-domain target hostname pattern:
- FAH publishes a target like `<random-id>--<backend-name>.<region>.hosted.app`
- Cannot be predicted — must be read from FAH after backend creation
- Hence the dependency: Plan 15-04 (FAH backend) DEPENDS on this plan only at the "wire custom domain" step. We pre-create a placeholder DNS record here using the pattern from admin's existing record, and adjust in Plan 15-04 if FAH publishes a different target.

Reference admin's existing DNS record (read-only, do not modify):
```bash
mcp__godaddy__dns.records({ domain: "triarch.dev", type: "A", name: "admin" })
# OR
mcp__godaddy__dns.records({ domain: "triarch.dev", type: "CNAME", name: "admin" })
```
Mirror that record TYPE on the portal record. If admin uses A pointing at IP X, portal uses A pointing at IP X. If admin uses CNAME pointing at `<id>--admin.<region>.hosted.app`, portal will need a portal-specific CNAME target — but we can pre-publish a placeholder pointing at the SAME IP/CNAME admin uses; Firebase routes by Host header so the same target serves both backends correctly once they're both wired.
</reference>
