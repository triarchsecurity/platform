<objective>
Add `actor_source` column to `release_approvals` and ship a partial unique index that allows only ONE row with `decision='approved'` per `release_id`. The customer's approve flow already inserts exactly one such row; this constraint is defense-in-depth that prevents any future code path from creating duplicate approved rows for the same release. The `actor_source` column captures origin (`'web'` | `'slack'`) for audit-trail unification (Pitfall 4 in PITFALLS.md). Schema migration ships in the same deploy as the Web Promote button per the standing decision in STATE.md and Pitfall 3.

Purpose: Foundation for PROM-04 (system prevents double-promote race) and Pitfall 4 prevention (audit trail unification).
Output: Schema + migration + snapshot that drizzle-kit check accepts; column and index live on the dev cluster after `db:push` runs at deploy time.
</objective>
