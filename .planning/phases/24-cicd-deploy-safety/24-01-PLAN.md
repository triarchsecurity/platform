<objective>
Land the per-repo `verify-deploy-target` GitHub Actions job + `.github/deploy-targets.yml` lookup table in BOTH the admin (`triarch-dev`) and portal (`triarch-portal`) repos. The job runs after `quality-gate`, reads the lookup table, and FAILS the pipeline if `${{ github.repository }}` doesn't match the hard-coded `app_hosting_backend` literal in the `deploy:` job. Wire as a `needs:` dependency of `deploy:` so a mismatch blocks the call to `deploy-firebase.yml@v4`.

Purpose: CI-01 is the only real blast-radius gate (per RESEARCH §Q5: GCP cannot enforce per-backend IAM on Firebase App Hosting; project-level grants work on every backend in `triarch-dev-website`). Without verify-deploy-target, a copy-paste error in `app_hosting_backend:` deploys the wrong app to the wrong backend — irreversible until manual rollback.

Output:
- `.github/deploy-targets.yml` committed to both repos (single-key shape per resolved OQ-3, NO `-dev` env-suffix expansion)
- `verify-deploy-target` job in each repo's `ci-cd.yml`
- `deploy:` job's `needs:` updated to include `verify-deploy-target`
- Version bump in both `package.json` (admin patch, portal patch)
</objective>
