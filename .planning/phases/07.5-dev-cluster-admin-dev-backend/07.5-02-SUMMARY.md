### Task 2: db-migrate.yml new reusable workflow

**Commit:** `8bafdd0` — `feat(db-migrate): add reusable workflow replacing manual psql ritual (ENV-03 v4)`

**File:** `.github/workflows/db-migrate.yml` (147 lines)

**Triggers:** Both `workflow_call` and `workflow_dispatch` (CONTEXT D-19 — manual workflow_dispatch only for v2.0)

**Inputs:**
- `environment` (dev|prod, default dev)
- `firebase_project_id` (required)
- `consumer_repo` (required)
- `consumer_ref` (optional, defaults to env branch: dev→dev, prod→main)

**Secrets:**
- `FIREBASE_SA_KEY` (required — GCP service account for Firebase secret access)
- `GH_PAT` (optional — falls back to github.token for private repo checkout)

**Step sequence:**
1. Checkout consumer repo at env-appropriate ref
2. Set up Node.js 22 with GitHub Packages registry
3. `npm ci` (with NODE_AUTH_TOKEN for private packages)
4. `npm install -g firebase-tools`
5. `google-github-actions/auth@v2` with FIREBASE_SA_KEY
6. `google-github-actions/setup-gcloud@v2`
7. **read_secret** — reads `DATABASE_URL_DEV` (env=dev) or `DATABASE_URL` (env=prod) via `firebase apphosting:secrets:access`, pipes through `tr -d '\n'` (Pitfall 7), masks via `::add-mask::`, writes to GITHUB_OUTPUT
8. **drizzle-kit push** — `DATABASE_URL` from step output, `npx drizzle-kit push` with NO `continue-on-error` (Pattern 4)
9. **Migration summary** — `if: always()`, JOB_STATUS from env: block (Phase 02-02 rule), writes to GITHUB_STEP_SUMMARY
