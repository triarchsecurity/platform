## Known Follow-up

**NEXTAUTH_URL placeholder (RESEARCH §Pitfall 4):** `apphosting.yaml` currently has `NEXTAUTH_URL: https://admin-dev.triarch.dev`. This is a placeholder. The actual FAH default URL for the `admin-dev` backend follows the `<app>-dev.<region>.hosted.app` pattern. Mike captures the real URL when creating the backend (RUNBOOK B-1) and must update `apphosting.yaml` accordingly before Google OAuth will work on the dev backend. If NEXTAUTH_URL doesn't match the actual FAH URL, OAuth redirects will be rejected.
