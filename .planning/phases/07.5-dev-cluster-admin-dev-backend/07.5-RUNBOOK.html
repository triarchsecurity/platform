Stand up a non-prod environment for every Triarch-owned app: new CockroachDB cluster, per-project dev DBs, per-project <app>-dev Firebase App Hosting backends, env-aware shared-workflows v4, plus close out Phase 4's deferred UAT.

Dedicated dev cluster, separate from triarchdev-24092 (prod).

  1. Sign in to cockroachlabs.cloud
  2. Create Cluster → Basic tier (smallest, ~$0/mo for low usage)
  3. Cloud: Google Cloud, Region: us-east1 (parity with prod)
  4. Cluster name: triarchdev-dev (or accept Cockroach-generated suffix)
  5. Create SQL user; save the auto-generated password to your password manager (you'll need it for connection strings)
  6. Wait for cluster to provision (~5 min)
  7. Click the cluster → Connect → copy the connection string template (looks like postgresql://<user>:<pass>@triarchdev-dev-NNNN.j77.aws-us-east-1.cockroachlabs.cloud:26257/defaultdb?sslmode=verify-full)
Save these securely
  • Cluster name (e.g., triarchdev-dev-1234)
  • SQL username + password
  • Connection string template (replace defaultdb with each project's <app>_dev)
Verify: cockroach sql --url "<connection_string>" -e "SHOW DATABASES;" shows defaultdb + system. No errors.
