### Idempotency

Idempotency is provided by `CREATE DATABASE IF NOT EXISTS` — a CRDB native no-op when the database already exists. Re-running the script after a partial failure produces zero errors and leaves all pre-existing databases intact.

### No GRANT or USE Statements

Per CONTEXT D-07: owner = default admin role on the new cluster. Per-DB role differentiation is deferred to v3. No `USE <db>` statements — the script is intentionally cluster-level only.

### How Mike Applies It

After provisioning the cluster (RUNBOOK A-1), Mike runs:
```bash
cockroach sql --url "<dev-cluster-url>" -f scripts/provision-dev-dbs.sql
```
Then verifies with `SHOW DATABASES;` (the commented verification block at the end of the file).
