<objective>
Build `src/lib/fah-rollout.ts` — the runtime HTTP client that calls Firebase App Hosting's Rollouts REST API directly, mirroring the JWT-mint + token-exchange + raw-fetch pattern of `src/lib/github-app.ts`. This is the foundational layer for Phase 13: API routes (13-02) call into this module; the client island (13-03) never imports it.

Three exported functions:
1. `mintFahAccessToken(): Promise<string>` — signs an RS256 JWT with the SA private key via `jose.SignJWT`, exchanges it at `oauth2.googleapis.com/token` for a 1-hour OAuth2 access token, caches at module level for 50 minutes (matches `github-app.ts` TTL strategy); single-flight latch prevents concurrent mints.
2. `createFahRollout({ projectId, location, backendId, branch })` — POSTs to the FAH Rollouts create endpoint, returns `{ ok, rolloutName, state }` or `{ ok: false, error, status }`. Branch input validated against `/^[a-zA-Z0-9/_.\-]{1,256}$/` before any HTTP call (Pitfall 5 + security).
3. `getFahRolloutState(rolloutResourcePath)` — GETs the rollout resource, returns `{ ok, state, buildState?, errorMessage? }` or error. Used by 13-02's status polling route.

Per RESEARCH.md (Option C ACCEPTED): no `googleapis` SDK, no Firebase MCP, no GitHub Actions intermediary. Direct REST call mirrors what the FAH CLI does internally and what was manually verified working in Phase 7.5.

Two new direct dependencies are added: `swr@^2.4.1` (for 13-03 client polling) and `jose@^5` (currently transitive via next-auth at v4; this plan promotes it to a direct dep so this lib has a stable explicit version contract). Both per RESEARCH.md SUMMARY ("two new runtime dependencies and zero new dev dependencies").

Purpose: PREV-03 — the runtime layer that "calls Firebase App Hosting programmatic rollout API to deploy the selected branch onto the project's existing dev backend." Without this lib, no API route can dispatch a rollout.
Output: A typed, structured-error-returning, fully-tested HTTP client. Plan 13-02 imports it.
</objective>

<!-- jose v5 SignJWT (already used elsewhere transitively): -->

```typescript
import { SignJWT, importPKCS8 } from 'jose';

const privateKey = await importPKCS8(privateKeyPem, 'RS256');
const jwt = await new SignJWT({ scope: 'https://www.googleapis.com/auth/cloud-platform' })
  .setProtectedHeader({ alg: 'RS256', typ: 'JWT' })
  .setIssuer(saEmail)
  .setSubject(saEmail)
  .setAudience('https://oauth2.googleapis.com/token')
  .setIssuedAt()
  .setExpirationTime('1h')
  .sign(privateKey);
```

<!-- Google OAuth2 token exchange request: -->

```
POST https://oauth2.googleapis.com/token
Content-Type: application/x-www-form-urlencoded

grant_type=urn:ietf:params:oauth:grant-type:jwt-bearer&assertion=<signed_jwt>
```

Response: `{ access_token: string, expires_in: number, token_type: 'Bearer' }`
