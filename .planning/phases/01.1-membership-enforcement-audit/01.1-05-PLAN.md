<objective>
For every project-scoped DETAIL endpoint (release-logs/[id], bug-reports/[id], feature-requests/[id], plus reports/[id] if classified accordingly), update each handler to fetch the row first, then for non-staff verify membership against `row.project`. Non-members receive 404 — NOT 403 — to mirror the page-level no-leak pattern from Phase 1.

Purpose: Currently any signed-in non-staff user can hit `/api/platform/bug-reports/{any-id}` and read a bug from a project they don't belong to (and PATCH it, in the bugs case). This plan adds the per-row membership check using the row's `project` column.

Output: Each detail file has GET and PUT/PATCH handlers that: (1) call `requireSignedIn`, (2) fetch the row, (3) return 404 if missing OR if the non-staff caller isn't a member of `row.project`, (4) proceed normally otherwise.
</objective>
