# Phase 01.1 Plan 01: API Auth Helpers Summary

**One-liner:** Three role-aware access-control helpers added to api-auth.ts with deprecated requireAdmin alias preserving 32 existing callsites.

## What Was Built

`src/lib/api-auth.ts` now exports four symbols:

| Export | Shape | Purpose |
|--------|-------|---------|
| `requireSignedIn()` | `{error, session}` | Session-only gate ‚Äî "is somebody signed in" (renamed from requireAdmin) |
| `requireStaff()` | `{error, session, ctx}` | 401 if unauthenticated, 403 if not staff or DB unreachable |
| `requireMembership(projectKey)` | `{error, session, ctx}` | 401 if unauthenticated, staff bypass, 403 if no membership or DB unreachable |
| `requireAdmin` (alias) | same as `requireSignedIn` | `@deprecated` ‚Äî preserves 32 existing callsites during v1.14.x rollout |

## Helper Details

**`requireSignedIn()`** ‚Äî Functionally identical to the old `requireAdmin`. Checks `session?.user` exists; returns 401 if not. No role or membership lookup.

**`requireStaff()`** ‚Äî Calls `getCurrentUserContext(session)` from Phase 1's auth-context. If `ctx` is null (DB error) or `!ctx.isStaff`, returns 403. The fail-closed behavior on DB error is intentional: we cannot prove staff without a successful lookup.

**`requireMembership(projectKey)`** ‚Äî Calls `getCurrentUserContext(session)`. If `ctx` is null ‚Üí 403. If `ctx.isStaff` ‚Üí bypass success for all projects. Otherwise checks `ctx.memberships.some(m => m.project_key === projectKey)`; 403 if no match. Callers that want the 404-no-leak pattern (project existence not revealed) should translate the 403 themselves after fetching the row.

**`requireAdmin` (deprecated alias)** ‚Äî `export const requireAdmin = requireSignedIn`. Preserves the exact `{error, session}` return shape the 32 existing callsites destructure. The `@deprecated` JSDoc tells editors and linters to flag it; the v1.15 backlog item will remove it once Plans 03/04/05 have migrated every caller.

## Deprecated Alias ‚Äî Why It Must Stay

32 route files currently call `requireAdmin` and destructure `{error, session}`. Changing them all in this plan would make an already-large PR (access-control reclassification) unbounded in scope. The alias is the minimum viable backward-compat shim:
- Existing callers compile without modification
- TypeScript inference works: `requireAdmin()` returns `Promise<{error: NextResponse|null, session: Session|null}>`
- Each callsite can be individually migrated to `requireStaff` or `requireMembership` in Plans 03/04/05 with a clear, reviewable diff per endpoint class

## Migration Order for Plans 03/04/05

1. **Plan 03 ‚Äî Staff-only endpoints (batch):** Replace `requireAdmin` with `requireStaff` across ~20 endpoints that must never be accessible to non-staff. These are the blast-radius endpoints (project creation/destroy, provisioning, navigation CMS, settings, backfill).
2. **Plan 04 ‚Äî Project-list endpoints:** Replace `requireAdmin` with `requireSignedIn` + membership filter on the 3 list endpoints (release-logs, bug-reports, feature-requests GET). Non-staff gets filtered results, not 403.
3. **Plan 05 ‚Äî Project-detail endpoints:** Replace `requireAdmin` with `requireMembership` on the 6 detail endpoints (release-logs/[id], bug-reports/[id], feature-requests/[id] GET+PUT). Non-member ‚Üí 404 (no-leak pattern).
