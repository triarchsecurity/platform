<objective>
Create the `slack_action_audit` table (SCHEMA-02) — Drizzle schema declaration + generated migration 0011. This is the audit substrate consumed by Phase 7 (OttoBot dispatcher hardening): every Slack interactive button click writes one row capturing action_id, actor email, Slack user_id, payload hash, response status, dispatcher latency.

Purpose: SCHEMA-02 is a hard prerequisite for Phase 7 — without this table, the audit logger in `/api/slack/interact` cannot persist rows. Stand-alone immutable audit log; no foreign keys (per CONTEXT: audit logs are independent — actor_slack_id may not map to any other entity, action_id is a free-form identifier owned by the dispatcher).

Output: New `slackActionAudit` pgTable in `schema.ts`; generated migration 0011 with `CREATE TABLE` + `CREATE INDEX` on `created_at desc`. DB push deferred to Mike (App Hosting secret pattern, same as 03-01).
