<objective>
Add `release_logs.branch varchar(256) DEFAULT 'main'` column via Drizzle (SCHEMA-01). Backfill all existing rows to `'main'` (v1.14 only tracked single-branch flow). Update the ingest endpoint to accept an optional `branch` field so Phase 2 (shared-workflows) can start passing branch context once it ships — until then, omitted/null branch defaults to `'main'`.

Purpose: Phase 5 (customer page RC UI) groups releases by branch. Phase 6 (promoteAndAudit rewrite) writes branch context on dispatch. Both depend on this column existing. Single-task-flavored deliverable but split into two tasks: schema delta + migration generation (Task 1) and ingest endpoint update (Task 2).

Output: Updated `schema.ts` with `branch` column on `releaseLogs`, generated migration 0010 (ADD COLUMN + UPDATE backfill), and the ingest endpoint accepting optional `branch`. DB push deferred to Mike post-merge (DATABASE_URL is App Hosting secret — same precedent as v1.14 Phase 02-01 / 04-01).
